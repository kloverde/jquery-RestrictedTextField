The first rule of programming: never trust your data. If you're submitting a form, you still need to validate the input on the back end.

Capturing validation failures

This text field has a listener to capture the validationFailure and validationSuccess events. This text field has been configured to allow invalid text to remain in the field. The field's border color will be red for as long as the invalid input remains.

Alpha characters (A-Z, case-insensitive)

The remaining examples prevent invalid text from being entered at all, so although you could capture validationFailure and validationSuccess, you don't really need to - unless you're planning on calling attention to the fact that the user's input was ignored.
